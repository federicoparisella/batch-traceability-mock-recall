# Contributing

Thank you for considering contributing to this project! It's people like you that make open source tools great.

## Where to start?
If you are new to this project, check out the **Issues** tab.
Look for issues labeled **good first issue** or **help wanted**.
We currently have an open issue for **Code Quality Improvements & Refactoring** which is a great place to start!

## Getting Started

1. **Fork** the repository on GitHub.
2. **Clone** your fork locally:

    git clone https://github.com/federicoparisella/batch-traceability-mock-recall.git

3. **Install dependencies**:

    pip install -r requirements.txt

## Making Changes
1. Create a new branch for your feature or fix:

    git checkout -b feature/amazing-improvement

2. Make your changes in the code.
3. (Optional but recommended) Run tests to ensure nothing is broken:

    pytest

## Submitting a Pull Request (PR)
1. Push your changes to your fork.
2. Go to the original repository and click **"Compare & pull request"**.
3. Describe your changes clearly.
4. Wait for review!

Thank you!
